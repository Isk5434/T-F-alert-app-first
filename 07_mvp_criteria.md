# MVP受け入れ基準 (Acceptance Criteria)

PoCフェーズにおける品質・成果の合格ラインを定義する。

## 1. 機能要件 (Functional)

### A. 監視・検知
- [ ] JAAFおよび指定3県陸協のHP更新を、更新から **24時間以内** に検知できること。
- [ ] PDF形式の要項・リザルト内のテキストを抽出できること（画像化されたPDFを除く）。
- [ ] 差分検知において、ヘッダー/フッターの日付変更のみ等の「ノイズ」を90%以上除外できること。

### B. 通知・表示
- [ ] ユーザーが設定した「専門種目」「所属県」に関連する情報のみがフィルタリングされること。
- [ ] 通知詳細画面から、1タップで原文ページへ遷移できること。
- [ ] Push/LINE通知が、送信トリガーから **1分以内** にユーザー端末に届くこと。

### C. 共有
- [ ] 共有ボタンから発行されたURL/テキストを開いた第三者が、ログインなし（または簡易プレビュー）で情報を閲覧できること。

## 2. 非機能要件 (Non-Functional)

### Reliability (信頼性)
- [ ] **誤報率**: HIGH重要度の通知における「内容間違い」「リンク切れ」が **5%未満** であること。（PoC初期は10%許容）
- [ ] **稼働率**: システムが日中（8:00-22:00）に停止しないこと。

### Compliance (法務・倫理)
- [ ] `robots.txt` Disallow へのアクセスログが存在しないこと。
- [ ] 重大誤送信（事実無根の大会中止通知など）が **0件** であること。

## 3. ユーザーサクセス (User Success)

### Engagement
- [ ] **継続率**: モニターユーザー30名のうち、**50%以上** が4週間後も通知を開封またはアプリを起動していること。
- [ ] **反応率**: HIGH通知の **80%以上** が開封（詳細閲覧）されること。

### Qualitative
- [ ] アンケートで「このアプリがなかったら情報を見落としていた」という回答が **3件以上** 得られること。
