# GTM (Go-to-Market) / 初期ユーザー獲得案

ToC（競技者・コーチ）視点でのPoCユーザー獲得および初期グロース戦略。

## 1. ターゲットセグメントの優先順位
1. **大学陸上部 主務・マネージャー** (情報感度・責任感が最も高い / 拡散力あり)
2. **クラブチーム・高校部活の熱心なコーチ** (ITリテラシーが高めの層)
3. **個人登録のマスターズ陸上選手** (自分で手続きを行うためニーズが高い)

## 2. 獲得チャネルとアクション

### A. ダイレクトアプローチ（実業団・大学）
- **対象**: 関東学連所属大学の主務、実業団チームのマネージャー。
- **手法**: 「事務作業（要項チェック）を自動化しませんか？」という切り口でDM/問い合わせ。
- **オファー**: PoC期間中の無料利用権 ＋ 要望機能の優先実装権。

### B. コミュニティマーケティング（コーチ）
- **対象**: X (旧Twitter) や note で発信している陸上指導者。
- **手法**: "陸上DX" や "部活効率化" の文脈で、開発ブログやデモ動画を投稿し、リプライ等でベータテスターを募集。
- **コンテンツ**: 「要項変更に気づかずエントリー漏れ...を防ぐツールの開発日誌」等の共感を呼ぶストーリー。

### C. 大会会場でのゲリラ活動（まずは友人の範囲で）
- **対象**: 地域の記録会に参加している選手・コーチ。
- **手法**: 待機時間にチラシ/QRコード配布。「今の陸協HP見にくくないですか？」という現場のペインに直接訴求。

## 3. オンボーディングプロセス (PoC版)
1. **Web LP登録**: メールアドレス/LINE登録。
2. **ヒアリング**: 専門種目・所属・現在の情報収集方法（課題）をフォーム入力。
3. **招待**: テスト版アプリ（またはLINE Bot）のリンクを送付。
4. **定着支援**: 最初の1週間は、何か通知が届くたびに「役立ちましたか？」の簡易アンケートを実施。

## 4. 成長ループ（Viral Loop）の設計
- **共有からの流入**: 「コーチへの共有」機能を活用。コーチに通知が届いた際、コーチ自身も「便利だ」と感じて登録するフローを作る。
- **チーム導入**: 1人の主務が導入 → 部員全員にID配布（または共有閲覧）の流れを作る。
